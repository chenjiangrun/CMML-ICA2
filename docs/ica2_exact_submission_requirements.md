# ICA2 Exact Submission Requirements

Source files are archived in `assignment/source_requirements/`.

## Individual Work

- The mini-project report must be composed individually.
- Reports must not be shared with other students.
- Results, discussion, figures, and tables should not be identical or highly
  similar to other students' submissions.
- Code from public sources can be adopted, but it must be acknowledged clearly
  in the report.

## Main Report

- One report is required for each mini-project.
- The report contributes 50% of the final course mark.
- Style should resemble a short paper or Nature Methods brief communication.
- The main report should include:
  - Abstract: maximum 70 words, unreferenced.
  - Main text: maximum 1000 words, excluding figure/table legends.
  - Main text sections should cover background, aims, results, a brief methods
    explanation, and discussion.
  - Display items: maximum two figures and/or tables in the main report.
  - References: recommended maximum of 20 references.
- Word count for the main text and supporting materials should be provided.

## Supporting Materials

- Code/documentation may be submitted as a Jupyter notebook or GitHub
  repository. If using a repository, include the link in the Methods section.
- Supplementary Methods: maximum 500 words.
- Reflection: maximum 200 words. It should discuss action points, autonomous
  work, course-content connections, and future development.
- Supplementary Figures/Tables: optional; maximum five items. Every
  supplementary figure/table needs a legend and must be cited in the main text.

## Marking Criteria Highlighted in the Screenshots

- Understanding of topic and research methods.
- Concise, accurate abstract and main text.
- Logical flow from background to results and discussion.
- Strong results and discussion supported by high-quality evidence.
- High-quality labelled figures/tables that support the findings.
- Comprehensive, relevant, correctly formatted references.
- Well-organised supplementary methods.
- Functional, documented, reproducible code.
- Reflection showing engagement, critical thinking, and growth.
